## 🛡️ What CCMA does

CCMA runs a structured chain of checks around code generation. Each step has a job:

- **Claude Code** writes the code
- **Bash Guard** checks shell commands for risky actions
- **File Guard** checks file changes before they go through
- **Automatic rework loops** send weak code back for another pass
- **Security auditor** looks for unsafe patterns and common mistakes

This setup helps catch issues early. It also reduces the need to read every line by hand.

## 📁 How the pipeline works

A simple run usually follows this order:

1. You give CCMA a task
2. Claude Code writes the first version
3. Bash Guard checks command use
4. File Guard checks file edits
5. The rework loop looks for weak spots
6. The security auditor reviews the result
7. CCMA sends back the final output

This process keeps the work in steps. That makes it easier to spot problems before they spread through the project.
